### P2-C: Confirm-delete inline state in 3 components

**Problem**: `ContentItem.tsx`, `ContentCard.tsx`, and `RecordDetail.tsx` each implement a click-once-to-arm, click-again-to-confirm delete pattern using `confirmDelete`/`setConfirmDelete` state (~9 lines of state + handler per file). Each renders a slightly different UI for the armed state.

**Impact**: Three implementations to keep in sync. Minor — the pattern is simple enough that drift is low risk.

**Effort**: S — extract `useConfirmAction(onConfirm)` → `{ armed, arm, cancel, trigger }`. But the UIs differ enough that full sharing is tricky.

**Proposal**: Extract only the state/logic part as a small hook:
```ts
// frontend/hooks/useConfirmAction.ts
export function useConfirmAction(onConfirm: () => void) {
  const [armed, setArmed] = useState(false);
  const arm = () => setArmed(true);
  const cancel = () => setArmed(false);
  const trigger = () => { setArmed(false); onConfirm(); };
  return { armed, arm, cancel, trigger };
}
```

Each component keeps its own JSX rendering but replaces its 9-line state block with one `useConfirmAction` call.

**Dependencies**: None.
